## 一、AI漫剧是什么

### 1.1 定义与特点

**AI漫剧（AI Dynamic Comic / Motion Comic）** 是介于传统漫画与动画之间的内容形态——以AI生成的静态画面为基础，通过2.5D运镜、局部微动效、配音音效等手段，使其具有"动起来"的视觉效果，但仍保留漫画的画风和分镜语言。

核心特点：
- **AI生成画面**：角色、场景、分镜均由AI（Midjourney/SD/DALL·E）生成
- **动态化处理**：运镜（推/拉/摇/移）+ 局部动效（眨眼/口型/飘发）+ 转场特效
- **配音驱动**：TTS合成配音 + 音效/SFX + BGM，配音时长决定镜头节奏
- **竖屏优先**：主流平台（抖音、快手）以9:16竖屏为主，横屏（16:9）用于B站/YouTube

### 1.2 与纯漫画、动画的区别

| 维度 | 纯漫画 | AI漫剧 | 传统动画 |
|------|--------|--------|----------|
| 画面 | 静态插画 | AI静态图+运镜动效 | 逐帧手绘/3D渲染 |
| 角色一致性 | 人工绘制 | LoRA/IP-Adapter/种子管理 | 原画师统一设计 |
| 制作周期 | 数周-数月 | 数天-数周 | 数月-数年 |
| 成本 | 中（人力） | 低（GPU+工具） | 高（团队+场地） |
| 动态效果 | 无/简单翻页 | 2.5D运镜+微动效 | 全帧动画 |
| 配音 | 无/气泡文字 | TTS+音效+BGM | 专业声优 |

### 1.3 主流平台

| 平台 | 比例 | 时长偏好 | 特点 |
|------|------|----------|------|
| 抖音漫剧 | 9:16竖屏 | 1-3分钟/集 | 算法推荐，短平快，付费解锁 |
| 快手漫剧 | 9:16竖屏 | 1-2分钟/集 | 下沉市场，付费转化高 |
| B站漫剧 | 16:9横屏 | 3-8分钟/集 | 弹幕文化，长篇连载 |
| YouTube Shorts | 9:16竖屏 | <60秒 | 全球分发，AdSense变现 |
| 腾讯视频/爱奇艺 | 16:9横屏 | 5-10分钟/集 | 付费会员，精品化 |

## 二、故事板制作流程

### 2.1 从小说/剧本到故事板的完整转换

六步工作流：

```
Step 1: 剧本分析
  ├─ 提取关键场景、角色、动作
  ├─ 标注情绪曲线（起承转合）
  └─ 输出：剧本分析表

Step 2: 分镜拆解
  ├─ 按镜头切分（平均3-8秒/镜头）
  ├─ 标注景别 + 运镜 + 时长
  └─ 输出：分镜脚本表

Step 3: 提示词工程
  ├─ 组装场景提示词 + 角色提示词
  ├─ 注入风格参考
  └─ 输出：提示词批次文件 (JSON/CSV)

Step 4: 批量生成
  ├─ ComfyUI / Automatic1111 批量运行
  ├─ 每镜头生成3-5个候选
  └─ 输出：候选图集

Step 5: 筛选与排序
  ├─ 按构图/表情/一致性评分
  ├─ 选中最佳 → 标记镜头编号
  └─ 输出：初版故事板

Step 6: 后期合成
  ├─ 排版（分镜板格式）
  ├─ 标注对白 + 音效
  ├─ 加入时间轴标记
  └─ 输出：完整故事板 PDF/PPT
```

### 2.2 分镜设计要点

**景别与情绪对应：**

| 景别 | 英文 | 情绪/功能 | AI提示词关键词 |
|------|------|-----------|---------------|
| 大远景 | EWS | 渺小、壮阔、孤独 | extreme wide shot, tiny figure against vast landscape |
| 远景 | WS | 交代环境 | wide shot, character standing in the middle of the room |
| 全景 | FS | 完整人物 | full body shot, character walking confidently |
| 中景 | MS | 对话、日常互动 | medium shot, two characters talking, waist up |
| 中近景 | MCU | 情绪过渡 | medium close-up, chest up, character showing concern |
| 特写 | CU | 情感爆发 | close-up on face, tears streaming down |
| 大特写 | ECU | 极度紧张 | extreme close-up on eyes, pupil dilating |

**镜头角度：**

| 角度 | 情绪效果 | AI提示词 |
|------|----------|---------|
| 平视 | 中立、平等 | eye level shot, neutral perspective |
| 俯视 | 弱势、渺小 | high angle shot, looking down at character |
| 仰视 | 强大、压迫 | low angle shot, looking up at towering figure |
| 鸟瞰 | 上帝视角 | bird's eye view, top down |
| 荷兰角 | 不安、混乱 | dutch angle shot, tilted camera |
| POV | 代入感 | POV shot, seeing through character's eyes |

**运镜方式与情绪匹配：**

| 情绪 | 推荐运镜 | 速度 | 景别 |
|------|---------|------|------|
| 平静/安宁 | 静止/慢摇 | 极慢 | 全景 |
| 紧张/悬疑 | 慢推 | 缓慢 | 近景→特写 |
| 震惊/冲击 | 快推 | 快速 | 中景→特写 |
| 孤独/渺小 | 拉远 | 缓慢 | 近景→全景 |
| 追逐/行动 | 跟踪/甩镜 | 快速 | 中景 |
| 宏大/史诗 | 升降/环绕 | 缓慢 | 远景 |

**节奏设计原则：**

- **快节奏**（动作/紧张）：1-3秒/镜头
- **中节奏**（对话/叙事）：3-6秒/镜头
- **慢节奏**（抒情/氛围）：6-12秒/镜头

**景别变化节奏：**
- 递增式：WS→MS→CU→ECU（逐渐聚焦，制造紧张）
- 递减式：ECU→CU→MS→WS（逐渐拉开，制造释放）
- 跳跃式：WS→ECU（强烈视觉冲击）

### 2.3 关键帧提取策略

从小说中提取关键帧的核心方法：

1. **情绪转折点**：每次情绪发生质变时的画面（如从平静→愤怒）
2. **动作高潮点**：打斗/追逐/揭示的关键瞬间
3. **对话关键帧**：每段对话的起始（建立镜头）+ 核心台词（特写反应）
4. **场景转换点**：每场戏的开始（建立镜头）+ 结束（空镜/拉远）
5. **信息密度**：每3-5秒必须有一个视觉变化点

### 2.4 场景切换逻辑

- **匹配剪辑**：利用相似构图/色彩/形状做转场（如从圆形窗户切到圆形月亮）
- **空镜过渡**：在场景切换间插入空镜（风景/物品特写）缓冲
- **甩镜转场**：快速甩镜从一个场景切到另一个（动作场景常用）
- **色彩过渡**：利用色调渐变实现场景转换（暖→冷表示时间流逝）

## 三、AIGC工具链

### 3.1 故事板生成工具矩阵

| 工具 | 用途 | 优势 | 局限 |
|------|------|------|------|
| Midjourney V6 | 角色概念探索、场景生成 | 风格化强，出图质量高 | 角色一致性弱，无ControlNet |
| Stable Diffusion XL | 精细控制、批量生成 | ControlNet/IP-Adapter生态丰富 | 需要调参经验 |
| ComfyUI | 工作流编排、批量生产 | 节点式管理，可复用 | 学习曲线陡峭 |
| DALL·E 3 | 快速原型 | 理解自然语言好 | 风格化弱 |
| Fooocus | 一键出图 | SD简化版，上手快 | 控制能力有限 |

### 3.2 角色一致性方案（核心难点）

**方案一：LoRA微调（推荐，质量最高）**
- 收集10-20张同一角色多角度、多表情图
- Kohya SS / AI Toolkit 训练（约30分钟）
- 生成时调用：`<lora:char_name:0.8>`
- 优点：跨场景/跨姿势/跨表情高度一致
- 缺点：需要训练时间，角色变化需重新训练

**方案二：IP-Adapter / InstantID**
- 输入1张角色面部参考图即可
- IP-Adapter Face ID 保持面部一致性
- 适合不需要训练的场景
- 优点：即用即生效，灵活度高
- 缺点：复杂场景下可能不够稳定

**方案三：Midjourney 角色参考（--cref）**
- MJ V6 支持 `--cref URL` 参数
- 配合 `--cw 100`（特征权重）使用
- 优点：简单快捷
- 缺点：复杂场景可能不稳定

**方案四：ComfyUI 工作流**
- 串联 IP-Adapter + ControlNet + LoRA
- 一次性输出同一角色的多角度、多表情
- 可保存为JSON模板反复使用

### 3.3 背景生成方法

- **场景种子库**：为每个场景分配固定seed范围，相近种子保持色调统一
- **IP-Adapter风格参考**：用1张"场景风格参考图"注入所有生成
- **场景风格LoRA**：训练20张同风格场景图的LoRA，权重0.6~0.8
- **ControlNet引导**：MLSD控制建筑透视，Depth Map控制空间关系
- **后期统一调色**：Lightroom/DaVinci Resolve批量套用LUT

### 3.4 对话气泡/字幕嵌入

- **自动字幕**：配音生成后，使用Whisper/VITS提取时间轴，叠加字幕
- **气泡生成**：Midjourney/SD生成含对话气泡的漫画格（prompt加`speech bubble with text`）
- **后期叠加**：AE/PR/Sony Vegas在时间轴上精确叠加字幕

## 四、动态化制作

### 4.1 静态图→动态效果

**2.5D运镜（Parallax Camera Movement）：**
- 将静态图拆分为前景/中景/背景多层
- 使用After Effects Camera工具或Toonflow运镜库
- 模拟推/拉/摇/移/环绕等真实镜头运动
- 核心：深度图生成（Depth Map）+ 视差合成

**面部动画：**
- **Hedra**：照片级口型同步，输入图片和音频即可
- **Runway Gen-3**：面部微表情驱动
- **Pika**：简单面部动画，眨眼/微笑
- **即梦**：国内工具，支持角色口型同步

**局部微动效：**
- 头发飘动、衣物摆动、粒子效果（烟雾/光晕/雨滴）
- AE粒子系统 / After Effects 摄像机 + 模糊层
- Runway / Pika 做局部区域动画（inpainting模式）

### 4.2 常用动效工具

| 工具 | 功能 | 适用场景 |
|------|------|---------|
| After Effects | 专业级运镜+动效 | 精品漫剧、精细控制 |
| Runway Gen-3 | AI视频生成 | 全身动作、场景变化 |
| Pika Labs | AI局部动画 | 面部/头发/衣物微动 |
| Hedra | 口型同步 | 对话场景配音对口型 |
| 即梦（Dreamina） | 国内AI动画 | 快速出片，中文友好 |
| ComfyUI + AnimateDiff | 批量生成短视频 | 批量生成角色动作片段 |
| EbSynth | 风格帧插值 | 关键帧驱动中间帧 |

### 4.3 配音与音效同步

**配音流程：**
1. **TTS生成**：使用ElevenLabs/硅基流动/魔音工坊生成配音
2. **时间轴对齐**：根据配音时长反推镜头时长
3. **口型同步**：Hedra/Runway做口型匹配
4. **BGM+音效**：在配音间隙插入环境音/特效音/背景音乐

**节奏对齐原则：**
- 配音时长 = 镜头时长（含前后各0.5秒缓冲）
- 每句台词对应一个镜头或镜头组
- 重要台词配合特写/慢推，增强戏剧性

## 五、对我们工具的启发

### 5.1 现有资产与漫剧管线的契合度

**已有的 Toonflow 素材库已覆盖漫剧核心需求：**
- ✅ 角色素材库（4视图+6配饰+5表情+配色方案）
- ✅ 运镜动作素材库（8种运镜+4种景别+角色动作）
- ✅ 场景提示词库（古风/自然/室内/奇幻/夜景）
- ✅ 提示词模板集（角色/运镜/场景/分镜模板）

**已有的 html-video + 小说管线可支撑：**
- ✅ 小说→剧本→分镜的文本管线
- ✅ HTML视频导出（基础播放/字幕）

### 5.2 需要补充的能力

**优先级P0（必须）：**

1. **角色一致性生成管线**
   - 集成 LoRA 训练模块（Kohya SS API 封装）
   - 或 IP-Adapter / InstantID 集成（SDXL生态）
   - 角色板自动排版导出

2. **批量分镜生成器**
   - JSON批次文件解析 → ComfyUI/SD API调用
   - 每镜头多候选生成 + 自动评分筛选
   - 提示词模板引擎（从分镜JSON自动生成提示词）

3. **2.5D运镜合成器**
   - 深度图生成（MiDaS/ZoeDepth）
   - 视差图层拆分 + Camera Move合成
   - Toonflow运镜库扩展（对接已有M001-M007运镜）

4. **配音同步模块**
   - TTS集成（硅基流动/魔音工坊API）
   - 时间轴自动生成（配音时长→镜头时长映射）
   - 字幕文件生成（SRT/ASS）

**优先级P1（重要）：**

5. **口型同步模块**
   - Hedra/Runway API集成
   - 对话场景自动口型驱动

6. **音效/BGM管线**
   - AI音效生成（AIVA/Epidemic Sound API）
   - 自动音效匹配（场景→音效标签映射）

7. **平台导出配置**
   - 抖音9:16竖屏模板
   - B站16:9横屏模板
   - 分辨率/码率/帧率预设

### 5.3 推荐的实现路径

**Phase 1（MVP，2-3周）：**
```
小说 → 分镜JSON → 提示词生成 → SD批量出图 → 静态HTML视频
```
- 复用现有小说管线 + 提示词模板引擎
- 实现SD API批量调用（替代部分Toonflow功能）
- HTML视频增加基础字幕

**Phase 2（运镜+配音，3-4周）：**
```
静态图 → 深度图 → 视差运镜 → TTS配音 → 合成视频
```
- 集成MiDaS深度图生成
- 实现2.5D运镜合成（基于Toonflow运镜库）
- 接入TTS API + 字幕同步

**Phase 3（口型+音效+导出，2-3周）：**
```
运镜视频 → 口型同步 → 音效叠加 → 平台导出
```
- Hedra/Runway口型同步
- 自动音效匹配
- 多平台导出配置

**Phase 4（角色一致性，持续优化）：**
- LoRA训练管线集成
- IP-Adapter工作流封装
- 角色一致性自动检测
