# 小说创作核心 Skill

> 定义 AI Agent 写小说的标准化流程：大纲→章节→段落→校对

## 1. 总览

本 skill 定义了一个四步流水线，适用于任何题材。每个步骤有独立的 prompt template 和校验规则。

```
用户输入（题材+字数+风格）
    │
    ▼
[Step 1] 大纲生成 → 三幕结构 + 人物表 + 世界观
    │
    ▼
[Step 2] 章节拆解 → 每章目标 + 关键情节点
    │
    ▼
[Step 3] 段落写作 → 逐段生成，流式输出
    │
    ▼
[Step 4] 校对润色 → 6道门禁 + 风格一致性
    │
    ▼
最终输出（章节文本 + 状态更新）
```

---

## 2. Step 1: 大纲生成

### Prompt Template

```
你是一个专业的{题材}小说作家。请为以下输入生成完整大纲：

【题材】{genre}
【目标字数】{word_count}
【风格】{style}
【附加要求】{extra_requirements}

请按以下格式输出：

## 一、三幕结构

### 第一幕（0-25%）
- 开场场景：
- 主角状态：
- 核心冲突引入：
- 关键情节点①（25%处）：

### 第二幕（25-75%）
- 发展阶段：
- 关键情节点②（50%处）：
- 关键情节点③（75%处）：
- 角色弧光转折：

### 第三幕（75-100%）
- 高潮场景：
- 关键情节点④：
- 结局：

## 二、人物设定
| 角色 | 年龄 | 身份 | 核心动机 | 弧光方向 |
|------|------|------|----------|----------|
| ... | ... | ... | ... | ... |

## 三、世界观
- 时间背景：
- 空间背景：
- 核心规则：
- 势力分布：

## 四、核心主题
- 主题1：
- 主题2：
```

### 校验规则
- ✅ 三幕结构完整，每幕有明确的情节点
- ✅ 主要角色≥3个，有明确动机
- ✅ 世界观设定不矛盾
- ⚠️ 字数建议是否合理（大纲篇幅≈目标字数的1-2%）

## 3. Step 2: 章节拆解

### Prompt Template

```
基于以下大纲，将故事拆解为章节：

{outline}

【总字数】{total_word_count}
【每章字数】{chapter_word_count}

请按以下格式输出：

## 章节列表

| 章节 | 标题 | 字数 | 核心内容 | 情节点 | 出场角色 |
|------|------|------|----------|--------|----------|
| 第1章 | ... | 3000 | ... | ... | ... |
| 第2章 | ... | 3000 | ... | ... | ... |
...

## 关键标记
- ⭐ 高潮章节
- 📖 信息章节
- 🔥 冲突章节
- 💔 情感章节
```

### 校验规则
- ✅ 章节数合理（3000字短篇≈8-12章，10万字长篇≈30-40章）
- ✅ 每个情节点落在对应的章节中
- ✅ 所有角色在适当的章节出场
- ⚠️ 章节节奏：不能连续3章以上无冲突

---

## 4. Step 3: 段落写作

### Prompt Template

```
你正在写小说的第{chapter_number}章《{chapter_title}》。

【本章目标】{chapter_goal}
【出场角色】{characters}
【前情提要】{previous_summary}
【事实快照】{facts_snapshot}

写作要求：
1. 每段300-500字
2. 段落间有自然过渡
3. 对话和描写交替出现
4. 每章末尾留有钩子（hook）
5. 严格按照事实快照，不引入未注册的新角色/地点

请从以下段落开始写：
{next_paragraph_prompt}
```

### 段落写作规则

| 规则 | 说明 |
|------|------|
| 场景切换 | 每换一个场景用空行分隔 |
| 对话格式 | `「对话内容」+ 动作/神态描写` |
| 内心独白 | 用*斜体*或`_下划线_`标记 |
| 时间跳跃 | 用`---`分隔不同时间段 |
| 段落长度 | 300-500字，不超800 |
| 钩子 | 每章末尾留悬念或问题 |

## 5. Step 4: 校对润色

### 6道门禁校验

见 `agent-hub/gatekeeper.js` 的具体实现。以下是 prompt-level 的校对指令：

```
请对以下章节文本进行校对润色：

{chapter_text}

检查项：
1. [一致性] 角色名称/属性是否与 facts.jsonl 一致
2. [连续性] 情节是否与前文连贯
3. [风格] 语言风格是否与设定一致
4. [冗余] 是否有重复描述
5. [AI味] 是否有过度修饰、空洞感叹、万能句式
6. [逻辑] 时间线/因果关系是否自洽

输出格式：
- ✅ 通过 / ❌ 不通过
- 修改建议（如果有）：
```

### 反AI味检测清单

| 特征 | 示例 | 替换方案 |
|------|------|----------|
| 过度副词 | "他轻轻地、缓缓地……" | 直接动作描写 |
| 万能感叹 | "这是多么……啊！" | 删掉或替换为具体感受 |
| 完美对话 | 每个角色说话都完整清晰 | 加入停顿、口吃、省略 |
| 过度描写 | 每个场景都写3句环境 | 只写关键细节 |
| 情感标签 | "他感到愤怒" | 用动作和对话表现 |
| 总结式结尾 | "这件事让他明白……" | 删掉，让读者自己体会 |

## 6. 题材Skill 调用规范

本核心 skill 在执行 Step 1（大纲生成）和 Step 3（段落写作）时，应加载对应题材的 Skill 文件：

```yaml
# 在 prompt 前加载
genre_skill: agent-hub/genre-skills/{genre_id}.md

# 注入到 prompt 的附加指令
genre_instructions: |
  - 遵循 {genre} 题材的写作规则
  - 使用 {genre} 的典型情节结构
  - 参考 {genre} 的角色模板
```

---

## 7. 协作模式

```
小宝 (CEO)           → 决策：分配题材、审核大纲
  ├─ 千问A3          → 研究：拆书分析、题材调研
  ├─ OpenCode         → 执行：写作、生成文件
  └─ 小聪            → 审核：门禁校验、风格检查
```

各角色职责：
- **小宝**: 调用 Step 1 和 Step 2，分配任务
- **千问A3**: 题材研究、Skill 文件维护
- **OpenCode**: 调用 Step 3 执行写作
- **小聪**: 调用 Step 4 执行校对和门禁
